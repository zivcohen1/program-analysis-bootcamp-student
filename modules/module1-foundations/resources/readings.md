# Module 1: Readings and References

## Required Reading
- [Static Program Analysis](https://cs.au.dk/~amoeller/spa/) - Chapter 1: Introduction (Free online textbook by Anders Moller and Michael Schwartzbach)

## Recommended Reading
- [A Few Billion Lines of Code Later](https://cacm.acm.org/magazines/2010/2/69354-a-few-billion-lines-of-code-later/fulltext) - Using static analysis at scale (Coverity experience report)
- [Lessons from Building Static Analysis Tools at Google](https://cacm.acm.org/magazines/2018/4/226371-lessons-from-building-static-analysis-tools-at-google/fulltext)

## Tool Documentation
- [ESLint Getting Started](https://eslint.org/docs/latest/use/getting-started)
- [SonarQube Documentation](https://docs.sonarqube.org/)
- [Pylint User Guide](https://pylint.readthedocs.io/)

## Case Studies
- **Ariane 5 (1996):** Integer overflow in flight software. [ESA Report](http://www.di.unito.it/~damiani/ariane5rep.html)
- **Heartbleed (2014):** OpenSSL buffer over-read. [heartbleed.com](https://heartbleed.com/)
- **Knight Capital (2012):** Dead code reactivation caused $440M loss in 45 minutes

## Foundational Concepts
- Rice's Theorem (1953): No algorithm can decide non-trivial properties of programs in general
- Dijkstra's principle: "Testing shows the presence, not the absence, of bugs"
